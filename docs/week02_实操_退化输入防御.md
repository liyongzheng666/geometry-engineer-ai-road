# 第 2 周实操：退化输入的防御性提示词

> 配套大纲：[A轨_12周总纲.md](A轨_12周总纲.md) 第 2 周
> 配套代码：[`code/week02_A/`](../code/week02_A/)（构建方法同 week01_A，见其 [README](../code/week02_A/README.md)）
>
> 周目标：训练 AI 写出能扛恶意输入的代码，沉淀"退化检查清单" prompt 模板。
> 第 1 周练的是"把数学说清楚"，这周练的是"把**你脑子里的工业实务**说清楚"——
> AI 不会自动想到粗采样选初值、射线穿顶点的 tie-breaking、行列式为零的含义。

---

## 为什么选这三个算法

| 算法 | 退化重灾区在哪 |
|---|---|
| 点到 B 样条曲线投影（Newton-Raphson） | 初值不好就发散；最近点在端点时迭代往域外跑；等距双解时结果不可复现 |
| 点在多边形内（射线法） | 射线穿顶点数两遍、与水平边共线、点恰在边上、凹多边形、顶点数不足 |
| Möller-Trumbore 射线-三角形 | 平行射线、零面积三角形（相对 eps 也归零的坑）、边/顶点命中的 watertight 问题、背面约定 |

每个 harness 都同时给你两类反馈：
- **数值判定**（PASS/FAIL）：用与被测实现**不同数学路径**的金标准双算对照；
- **观察题**（无标准答案）：打印 AI 代码在退化输入下的行为，问你"它声明过这个约定吗"。

## 周一-周二：点到 B 样条曲线投影（4h）

工作台：`test_point_inversion.cpp`。曲线求值 `curve_eval` / 导数 `curve_deriv` / `curve_deriv2`
是给定基础设施——**把它们的声明贴进 prompt**，让 AI 只写 Newton 迭代外壳。

用总纲第 2 周的模板发 prompt（要点）：

```text
任务：实现点 P 到 B 样条曲线 C(u) 的最近点求解（point inversion）。
请按以下逻辑生成代码：
1. 写出目标函数 f(u) = (C(u) - P) · C'(u) 及其导数
2. 用 Newton-Raphson 迭代求 f(u) = 0
3. 必须包含的退化处理：
   - 初值 u_0 选择策略（推荐：先粗粒度采样取最近）
   - 迭代步长过小时（|Δu| < tol_parameter）的早停
   - 迭代发散（|f| 不下降连续 3 次）时的 fallback
   - u 跑出参数域时的 clamp 策略
4. 容差参数 tol_distance / tol_parameter / max_iter 全部作为可配置入参
5. 函数签名：照抄 harness PASTE ZONE 上方的注释
```

**关键技巧**：你已经知道的工业实务（粗采样选初值），**必须自己写进 prompt**。
删掉第 3 条再发一次对比试试——多数 AI 会给一个裸 Newton，Test 3（越界）和 Test 6（预算不足）立刻翻车。

验证金标准怎么读：
- **Test 1（法向偏移构造）**：从曲线上取 C(u*)，沿法向偏移 0.25 构造 P——投影必须返回 u*。这是"由答案构造输入"，比对照任何参考实现都硬。
- **Test 2**：与密集采样 + 黄金分割的独立路径双算对照。
- **Test 5/6 观察题**：等距双解收敛到哪侧、迭代不足时返回什么——看 AI 有没有声明行为。

## 周三-周四：点在多边形内（4h）

工作台：`test_point_in_polygon.cpp`。用总纲第 2 周的模板（射线穿顶点 tie-breaking、
共线边跳过、点在边上的约定、非简单多边形行为、顶点数 < 3）。

四个测试多边形各打一个痛点：正方形（基线）、凹"飞镖"（凹口数 2 次交叉）、
菱形（左右顶点 y 恰好等于 query 的 y）、台阶形（射线与水平边共线）。

验证金标准：**winding number（atan2 角度和）双算对照**——与射线法完全不同的数学路径。
SVG 是网格采样染色图：绿色应严丝合缝填满内部，**任何红色大点 = 双算不一致处**，
直接告诉你 AI 的 tie-breaking 错在哪个区域。

**亲手加题**：把你代码库里最病态的多边形数据（如果有）填进 main() 试试。

## 周五：Möller-Trumbore（2h）

工作台：`test_ray_triangle.cpp`。这次**反着用 AI——先让它教你，再让它写码**：

```text
请详细解释 Möller-Trumbore 算法的几何推导：
1. 为什么不显式构造平面方程也能判定相交
2. 为什么算法不需要预归一化的法向量
3. 算法中哪一步的行列式如果为零意味着什么几何状态
4. 与基于平面方程的传统方法相比，数值上为什么更稳定
```

读懂后再让它写代码。**这是 AI 最高价值的场景**：你能验证它说的对不对，
所以可以放心借它的脑子加速学习。

验证金标准：
- 与平面求交 + 重心坐标两段法双算对照；
- **重构校验**：`orig + t·dir` 必须等于 `(1-u-v)A + uB + vC`——这条与任何实现路径无关；
- Test 6 有个特别阴的坑：三点全重合时**相对 eps 也会归零**，`<` 和 `<=` 一字之差就是 NaN 命中。
  看看 AI 的代码踩不踩（我们自己手写第一版就踩了）。

## 周末：抽出"退化检查清单"模板（2h）

把这周的共性抽进 [`prompt_library/`](../prompt_library/)，建议路径
`01_algorithm_design/degenerate_checklist.md`（模板正文见总纲第 2 周周末一节）：

按四类穷举——**拓扑退化**（共点/共线/共面/自相交）、**数值退化**（极小量/NaN/Inf）、
**参数域退化**（越界/闭合开放/重复度）、**配置退化**（非凸/非简单/非流形），
每条列：触发条件 → 检测方法 → 处理策略 → 验证用例。

更新 `failure_log.md`：这周至少 3 条 AI 在退化输入上翻车的样本。

## 常见踩坑

| 现象 | 原因 / 应对 |
|---|---|
| point_inversion 的 Test 1 不过但 Test 2 过 | AI 的 Newton 收敛了但精度不够（早停过松），或 tol 没接对 |
| Test 3（越界）返回 u 在域外 | AI 没做 clamp——你的 prompt 第 3 条写了吗？ |
| 多边形 Test 3/4 翻车 | tie-breaking 规则错。经典解法是半开区间 `(yi>y)!=(yj>y)`，让 AI 解释它的规则 |
| ray-triangle 在三点重合时返回 NaN | 相对 eps 归零 + `<` 比较。观察题专门埋了这个 |
| 双算不一致红点贴着多边形边界 | winding 在近边界数值不稳，SVG 已跳过 0.05 带；只关心远离边界的红点 |

## 最后一句

> 第 1 周证明了"AI 能写对正常路径"；
> 这一周你会发现：**退化路径写不写得对，完全取决于你的 prompt 有没有提**。
> 把你十年攒下的"病态输入清单"显性化成模板——这是 AI 替代不了的部分，
> 也是它最需要你喂给它的部分。
