## 设计哲学：为什么是"高度场三角网"

前三周是单点算法，这周是**完整流水线**。为了让它仍能"零依赖 + 一眼看图 + 由答案构造输入"，
`mesh.h` 用规则栅格高度场三角网（顶点 (i,j) 在 (i·dx, j·dy, z_ij)），而不是通用半边结构：

| 决策 | 理由 |
|---|---|
| 高度场三角网（而非半边） | 保留真实网格的邻域/面法向/开放边界，却能俯视渲染成 SVG 色图 |
| 折痕（两平面相交）作测试面 | 折痕角度已知 → 加固定种子噪声 → 平滑后回测，金标准无可辩驳 |
| 双边滤波（而非拉普拉斯） | 拉普拉斯会把折痕一起抹圆；双边靠 range 核"认得出"特征 |
| 法向 / 权重是"给定基础设施" | 你只写 `smooth_step` 更新外壳——聚焦本周真正的难点：边界、收敛、退化 |

> 在你的真实项目里，"高度场三角网"这一步对应把**半边结构头文件全文注入上下文**
> （第 3 周技巧）。接口语义一致，换汤不换药。

## 完整工作流（融合进第 4 周日程）

### 周一-周二：推双边滤波的数学（第 1 周技能）

用第 1 周的公式化 prompt（先要 LaTeX，再列参数几何意义）让 AI 写出**基于法向的双边网格平滑**公式：

```text
请按顺序回答，先分析后写码：
1. 写出保特征网格平滑（bilateral mesh denoising）顶点更新的 LaTeX：
   v' = v + n · ( Σ_j w_j·h_j / Σ_j w_j )，h_j=(v_j - v)·n
2. 逐项说明权重核 w_j = W_c(‖v_j-v‖)·W_s(h_j) 里每个参数（σc 空间、σs range）的几何意义
3. 解释：为什么 σs 小就能"卡住"折痕（跨折痕的 h_j 很大 → range 核趋零）
```

把 ② 得到的核粘进 `test_bilateral_weight.cpp` 验证。**Test 6 会亲眼给你看**：
σs=0.1 时跨折痕权重 1.8e-22，σs=1.0 时 0.57——保边旋钮的价值一目了然。

### 周三：注入网格结构，写法向 + 权重（第 3 周技能）

把 `mesh.h` 里 `Mesh` 的接口（`incident_tris` / `one_ring` / `tri_area_normal`）**贴进 prompt**，
让 AI 写 `vertex_normal`。强调两条退化：**零面积三角形不得产生 NaN**、高度场法向朝上。

跑 `test_vertex_normal`：Test 1 倾斜平面是**精确金标准**（平面网格无离散化误差，tol 1e-12）；
Test 2 抛物面只要求量级正确（离散法向天生有 O(h) 偏差）；Test 3 专门埋了零面积三角形。

### 周四：写顶点更新主循环（本周核心）

`vertex_normal` / `bilateral_weight` 在 `test_mesh_smooth.cpp` 里是**给定基础设施**——
把它们的签名贴进 prompt，让 AI 只写 `smooth_step` 外壳，特别要求：

- **开放边界**：边界顶点不更新（金标准 Test 3 要求位移严格为 0）
- **收敛判定 + 双终止**：迭代次数上限 + 位移/法向阈值，二者任一满足即停
- **退化兜底**：Σw≈0 的孤立顶点保持不动，不做 0 除

保特征三连金标准：① 平坦区 RMS 明显下降（去噪）② 折痕不下沉（`|ridge sink|`，
换成朴素拉普拉斯此项必崩）③ 边界零漂移。**Test 1 的说明还点破一个真相**：
双边滤波保特征的代价是留一层"噪声地板"，想压更低就得调大 σs，但那会抹圆折痕甚至发散——
这个取舍本身就是这周最值钱的体会。

### 周五：退化清单自查（第 2 周技能）

用第 2 周的退化清单 prompt 让 AI **自己**列出这个实现会崩的 10 种 mesh 输入：

```text
针对上面的 smooth_step，请按"拓扑/数值/参数域/配置"四类，列出至少 10 种会让它
崩溃或给出错误结果的网格输入（如：孤立顶点、退化三角形、全共面、尖刺噪声、
非流形边、超大量级坐标……）。对每种：触发条件 → 你的代码现在会怎样 → 该怎么兜底。
```

把 AI 没处理的补进 harness 的 `main()` 里亲手验证——这是把"AI 自查"变成"客观测试"的关键一跳。

## 这套流程的"杠杆"在哪里

1. **技能复用而非从零**：双边公式用第 1 周模板、网格注入用第 3 周技巧、退化自查用第 2 周清单——你会发现前三周的模板真的能拼成生产力。
2. **"保特征"是可度量的**：不是"看起来折痕还在"，而是 `ridge sink = 0.005`。双边 vs 拉普拉斯的差距变成一个数。
3. **σ 选择暴露判断力边界**：AI 会给你公式，但 σc/σs 取多少、噪声地板能接受多深——这些**工业容差决策**它给不了，正是你的护城河。

---

## 常见踩坑

| 现象 | 原因 / 应对 |
|---|---|
| vertex_normal Test 1 就 FAIL | 法向没归一化，或 cross 参数顺序反了（Test 4 会告诉你朝向翻没翻） |
| vertex_normal 中心顶点返回 NaN | 零面积三角形除了零。跳过零面积项，或对退化面积设阈值 |
| bilateral_weight Test 2 不过 | 少乘一个核，或指数里漏了 1/2 / 漏了平方 |
| smooth_step Test 1 不过但 Test 3 过 | 更新根本没生效（还是占位桩），或只动了边界 |
| 折痕被抹圆（ridge sink 变大） | σs 太大，range 核没能卡住折痕；或误用了拉普拉斯（没乘 range 核） |
| 平滑后网格"炸开"/发散 | σs 太大导致过松弛，沿法向过冲。调小 σs、加位移阈值早停 |
| 边界顶点动了（Test 3 FAIL） | 主循环没跳过 `is_boundary` 顶点 |
