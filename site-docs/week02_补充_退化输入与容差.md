# 第 2 周补充：退化输入分类学与容差哲学

> 这篇是第 2 周三个 harness 背后的"理论课"：退化输入怎么系统地穷举，
> eps 到底该怎么取。内容会直接变成你 prompt_library 里的
> `degenerate_checklist` 模板的弹药。

## 一、退化输入四分类

让 AI（或自己）穷举退化时，按这四类扫一遍，基本不会漏：

### 1. 拓扑退化 —— 元素间的关系塌缩

共点、共线、共面、自相交、零长度边、零面积三角形。
本周实例：三点共线的"多边形"、三点重合的"三角形"、
点恰好落在多边形顶点/边上。

### 2. 数值退化 —— 浮点本身的极端

量级悬殊（大数吃小数）、相近数相减（catastrophic cancellation）、
NaN/Inf 传播、subnormal。
本周实例：NaN 与任何数比较都是 false——所以 NaN 一旦产生，
所有 `if (x < 0) return fail;` 形式的防线**全部静默失效**。

### 3. 参数域退化 —— 定义域边界

参数越界、闭曲线 vs 开曲线、节点重复度、半开区间的右端点。
本周实例：point inversion 的 u 越界 clamp；第 1 周 B 样条 u=1.0 归零
也是这一类。

### 4. 配置退化 —— 算法前提不满足

非凸、非简单（自相交多边形）、非流形、顶点数不足。
本周实例：顶点数 < 3 的"多边形"、even-odd 还是 winding 的约定分歧。

> prompt 模板写法："对每条退化列出：触发条件 → 检测方法 → 处理策略 → 验证用例，
> 并在代码对应位置加行内注释 `// degen: <分类>`。"

## 二、容差哲学：eps 没有"正确值"，只有"正确量纲"

### 绝对 eps 的问题

```cpp
if (std::abs(det) < 1e-12) return degenerate;
```

输入单位是米还是毫米？模型缩放 1000 倍后这行代码的语义完全变了。
绝对 eps 隐含假设"输入已归一化"，而这个假设几乎从不成立。

### 相对 eps 的问题（本周的翻车现场）

```cpp
const double eps = 1e-12 * norm(e1) * norm(e2) * norm(dir);
if (std::abs(det) < eps) return degenerate;   // 三点全重合：0 < 0 为假！
```

相对 eps 量纲正确，但**参考量自己也会退化归零**。
三点全重合 ⇒ eps = 0 ⇒ 检测失效 ⇒ 除以零 ⇒ NaN。
两种修法：

```cpp
if (std::abs(det) <= eps) return degenerate;        // ① <= 把 0==0 收进来
if (edge_scale < kEps) return degenerate;            // ② 先单独拦"参考量为零"
if (std::abs(det) < kEps * edge_scale) return ...;
```

第 3 周的 circumcircle 用的是②（先拦重合点，再用相对 eps 拦共线）——
分两道闸，每道的语义都清晰。

### 经验法则

| 比较对象 | 建议 |
|---|---|
| 长度、距离 | 相对于输入包围盒尺度：`eps_rel * bbox_diag` |
| 面积类行列式 | 相对于边长乘积：`eps_rel * |e1||e2|` |
| 参数（u, t） | 可以用绝对 eps（参数域天然归一化到 [0,1]） |
| 与零比较且参考量可能为零 | `<=`，或先拦参考量为零的分支 |

**最重要的一条**：阈值是**接口契约**，不是实现细节。该写进函数注释、
写进 prompt、作为参数暴露（像 point_inversion 的 tol_distance / tol_parameter），
而不是埋在函数体里当魔法数。

## 三、tie-breaking：规则选得好，补丁全不要

射线法"穿顶点数两遍"的教科书修法是给顶点加特判——错的方向。
正确方向是换规则，让边界情况**自动落进**一般情况：

```cpp
// 半开区间规则：边只在 (yi > y) != (yj > y) 时计数
// = 每条边"包含下端点、不含上端点"
// → 射线穿顶点：共享该顶点的两条边里恰好只有一条计数
// → 射线与水平边共线：yi == yj，条件恒假，整条边自动跳过
```

类似的还有 Simulation of Simplicity（给输入加符号扰动）、
半开区间的 B 样条 span 闭合（第 1 周 u=1.0）。共同思想：
**找一个把所有边界 case 都覆盖的统一规则，胜过给每个 case 打补丁。**

## 四、给 prompt 的最后一句话

把这篇的内容压缩成一行加在你的几何算法 prompt 末尾，性价比极高：

```text
所有容差必须：写明量纲（绝对/相对于什么）、作为参数暴露、
并考虑参考量本身退化为零的情况。
```
