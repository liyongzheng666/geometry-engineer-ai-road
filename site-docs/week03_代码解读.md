## 一、mini_types.h：一份"假公司头文件"

要测"风格注入有没有生效"，得先有一套**风格信号足够强**的类型。
`gx::` 命名空间的每个设计决定都和 `geo.h` 反着来：

| 维度 | geo.h（AI 的舒适区） | gx::（注入的"项目风格"） |
|---|---|---|
| 运算 | `a + b` 运算符 | `a.add(b)` 成员函数 |
| 数据 | public `x, y, z` | private 数组 + `x()/y()/z()` 访问器 |
| 错误 | 无 | `GxStatus` 错误码 + 输出参数，禁异常 |
| 容差 | 字面魔法数 | 统一常量 `gx::kEpsGeom` |

差异越大，判分越容易：AI 偷懒写一个 `a + b`，编译直接报错；
偷偷 `throw`，肉眼一扫就见。`normalized()` 的签名本身就是教学：

```cpp
GxStatus normalized(Vec3& out) const;   // 零向量 → kDegenerateInput
                                        // 项目禁异常，错误走返回码
```

`mini_types.h` 刻意**不 include geo.h、不知道 Point3D 存在**——
维持"独立公司头文件"的人设。harness 里需要画 SVG 时，
用一个本地 `static Point3D to_p3(const gx::Vec3&)` 桥接，桥接代码属于
harness 基础设施，不属于 mini_types（公司头文件不该为测试工具改设计）。

## 二、test_circumcircle：数值层 + 风格层双重判分

### 数值层

三维外心的经典公式（PASTE ZONE 注释里直接给了，本周不考推导）：

```text
n = ab × ac
center = a + [ |ac|²(n×ab) + |ab|²(ac×n) ] / (2|n|²)
```

测试覆盖四个角度：

- **由答案构造**：圆心 (1,2,0)、r=2.5 上取 20°/100°/200° 三点，解必须复原；
- **与算法无关的性质**：圆心到三点等距（任何正确实现都满足，任何错误实现都难蒙混）；
- **解析特例**：直角三角形外心 = 斜边中点（手算即得 (2, 1.5)、r=2.5）；
- **三维倾斜圆**：法向 (1,1,1)/√3 的平面上取三点——**很多 AI 默认写 2D 公式**
  （只用 x/y），这条测试专杀这种偷懒。

参考实现是裸 double 的 2D 行列式闭式解，刻意**不用 gx 类型**——
既是独立数学路径，也避免与被测代码共享任何风格。

### 退化层

重合点和共线分**两道闸**检测（对应上一篇容差文章的方案②）：

```cpp
if (ab.norm() * ac.norm() < kEpsGeom)       return kDegenerateInput; // 重合
if (n.norm() < kEpsGeom * ab.norm()*ac.norm()) return kDegenerateInput; // 共线
```

观察题（Test 7）给三点 `(0,0) (2,1e-9) (4,0)`——数学上 r≈2e9 的"合法"圆。
AI 接受还是拒绝？它的阈值是绝对的还是相对边长的？**它用了 kEpsGeom 吗**——
风格注入是否渗透到了容差这一层？

### 风格层

main() 末尾打印自查清单（数值全 PASS 也别跳过）：

```text
① PASTE ZONE 里有没有 std:: 容器？
② 有没有 a+b 这类运算符，而不是 a.add(b)？
③ 错误路径返回的 GxStatus 语义对不对？容差用的是 kEpsGeom 吗？
```

## 三、test_plane_fit：性能也能当判分项

### 接口即约束

```cpp
bool plane_fit(const double* points, int n,
               Eigen::Vector3d& centroid, Eigen::Vector3d& normal);
```

输入是裸 `const double*`（row-major N×3）——逼着实现方用
`Eigen::Map` 零拷贝接入，而不是先复制进一个 `std::vector<Vector3d>`。
工业代码里点云从来都是别人给的内存块，这个签名就是现实。

### 数学要点

最小二乘平面 = 质心 + 协方差矩阵最小特征方向：

```text
c   = mean(p_i)
cov = Σ (p_i - c)(p_i - c)ᵀ          ← 3×3，不管 N 多大
normal = cov 最小特征值对应的特征向量
```

**对 3×3 做分解，不是对 N×3**——这是大纲反复强调的点。
共线检测看第二小特征值：点集张不成面时 σ₁ ≈ σ₂ ≈ 0。

### 金标准

- **精确共面恢复**：构造平面（过 Q=(1,2,3)，法向 (1,2,2)/3）上的网格点，
  法向必须复原到 1e-9（允许反号，所以比的是 `1 - |normal·n0|`）；
- **独立参考**：手写 Cramer 法解 z=ax+by+c 的正规方程——完全不用 Eigen。
  注意它解的是 z-残差最小二乘，与正交最小二乘**只在精确共面数据上等价**，
  所以只用于 Test 1 对照（这种"参考实现的适用边界"必须写进注释）；
- **加噪容差**：固定种子 mt19937(42)、法向噪声 ±0.01、60 点——
  角偏差 < 0.01 rad。固定种子保证任何人任何机器跑出同一组数。

### 性能观察题（Test 7）

N=200000 计时一次调用。逐行累加协方差的实现 ≈ 0.3ms；
如果 AI 写了 `MatrixXd centered = P.rowwise() - c.transpose()`
这样的显式 N×3 临时矩阵，内存翻倍、耗时上一个量级。
**计时数字是客观证据**——"看起来都能跑"的两版代码，在这里拉开差距。
