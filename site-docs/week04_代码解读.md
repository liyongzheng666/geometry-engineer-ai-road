# 第 4 周代码解读：把"保特征"变成三个可判定的数

> 对应代码：[`code/week04_A/`](https://github.com/liyongzheng666/geometry-engineer-ai-road/tree/main/code/week04_A)
> 本周是阶段一的合体项目。harness 的设计重心是：**让"折痕有没有被保住"从主观感觉
> 变成一个客观数字**，同时把法向/权重下沉成"给定基础设施"，让你只写 `smooth_step`。

## 一、mesh.h：为什么是高度场三角网

通用网格平滑要半边结构，但那会立刻引入"数据结构学习"，冲淡本周主题。
`mesh.h` 用**规则栅格高度场**：顶点 (i,j) 固定在 (i·dx, j·dy, z_ij)，每个栅格单元
沿对角线剖成两个三角形。它同时满足三件事：

| 需求 | 高度场三角网怎么满足 |
|---|---|
| 是真三角网 | 有 `one_ring` 邻域、`tri_area_normal` 面法向、`is_boundary` 开放边界 |
| 能一眼看图 | `MeshSVG` 俯视按 z 染色，before/after 并排 |
| 能由答案构造输入 | 已知折痕高度场 → `add_deterministic_noise` 固定种子加噪 → 平滑后回测 |

绕序 `(v00,v10,v11)` / `(v00,v11,v01)` 是刻意的——两类三角形的 `cross(B-A,C-A)`
都朝 +z，保证高度场法向朝上。`test_vertex_normal` 的 Test 4 专门验证这条没被 AI 写反。

`add_deterministic_noise` 用 LCG 固定种子，且**边界不加噪但仍消耗一个随机数**——
这样即使你改了"边界是否加噪"的逻辑，随机序列也不错位，PASS/FAIL 永远可复现。

## 二、test_vertex_normal：用平面做"零误差金标准"

离散顶点法向对曲面天然有 O(h) 偏差，没法拿 1e-12 卡。但对**平面**——
所有三角形共面，面积加权法向是精确的：

```text
倾斜平面 z = a·x + b·y + c  →  每个顶点法向 ≡ normalize(-a, -b, 1)
```

所以 Test 1 用平面，tol 1e-12（和第 1 周 B 样条的 partition-of-unity 一个思路：
挑一个"离散公式恰好精确"的特例当硬对照）。Test 2 才用抛物面，只要求量级正确，
并在输出里点明"这偏差是离散化本身，不是 bug"。Test 3 把中心顶点塌到邻居上制造零面积
三角形，判 `std::isfinite`——**AI 最容易在这里写出 0 除 → NaN 传染整条法向**。

## 三、test_bilateral_weight：核的代数性质当金标准

权重核 `w = exp(-d_c²/2σc²)·exp(-d_s²/2σs²)` 没有"参考数据"可对，但有一堆
**任何正确实现都必须满足、任何错误实现都难同时满足**的代数性质：

- **可分离性**（Test 2）：与"两个独立一维高斯之积"双算对照，tol 1e-12——
  参考实现 `gauss_ref` 走的是"分别算再相乘"的另一条书写路径；
- 非负 / ≤1 / 单调衰减 / 偶对称（Test 3–5）——漏乘一个核、指数漏 1/2 或漏平方都会破功；
- **Test 6 不是判分，是给直觉**：同样跨折痕（d_s=1.0），σs=0.1 权重 1.8e-22、
  σs=1.0 权重 0.57。σs 就是"多大的信号跳变算特征"的旋钮，这一眼值一段推导。

## 四、test_mesh_smooth：保特征三连 + 一条被点破的真相

`vertex_normal` / `bilateral_weight` 在这里是**给定基础设施**（`[[maybe_unused]] static`，
占位桩没调用它们时不刷警告）。PASTE ZONE 只剩 `smooth_step` 外壳，考点集中在
边界/收敛/退化。金标准三连：

```text
Test 1  平坦区 RMS：after < 0.6 × before      → 去噪有效（无操作占位桩 ratio=1.0 必失败）
Test 2  折痕下沉量 |ridge sink| < 0.12         → 特征保住（换朴素拉普拉斯此项必崩）
Test 3  边界顶点位移 == 0                       → 开放边界不漂移
```

**Test 1 的说明刻意点破一个真相**：双边滤波保特征的代价是留一层"噪声地板"——
σs 小到能卡住折痕时，平坦区的噪声（量级和 σs 相当）也被部分当成了"特征"而保留，
所以 RMS 只降到 ~50% 就到底。想压更低必须调大 σs，但那会同时抹圆折痕、甚至让沿法向的
过松弛更新发散。**这个取舍不是 bug，是双边滤波的本性**，也是本周最该带走的判断。

Test 4 是收敛观察题：连续几步的总位移应递减（8.3 → 2.3 → 1.1 → 0.6…）。
它不判分，但逼你回答 prompt 里那句——AI 有没有做"迭代次数 + 位移阈值"的双终止？发散怎么兜底？

## 五、本周的元规律

前三周每周练一招；这周证明这些招能**拼起来**：第 1 周模板推公式、第 3 周注入喂结构、
第 2 周清单做退化自查。而三个金标准把"算子好不好"压成了三个数——
但那个最关键的决定（σ 取多少、噪声地板忍多深）依然只有你能拍板。
**能自动化的都自动化了，剩下的正好是几何工程师的判断力。**
