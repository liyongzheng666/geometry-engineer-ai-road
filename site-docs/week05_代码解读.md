# 第 5 周代码解读：让有限差分给 SymPy 当裁判

> 对应代码：[`code/week05_A/`](https://github.com/liyongzheng666/geometry-engineer-ai-road/tree/main/code/week05_A)
> 本周把 AI 当符号推导搭档，但验证那一环焊死在 C++ 里：
> **SymPy 生成解析式，中心差分独立复算，两条数值路径对撞。**

## 一、energy.h：为什么是"2 变量弹簧能量"

选能量模型时有个张力：太简单（如 x²+y²）暴露不了链式求导的坑，太复杂（如网格 ARAP）
有限差分参考不可信、也没法画图。弹簧能量 `E=Σ½k(‖p−a‖−L)²` 正好卡在中间：

- **只有 2 个变量** → 中心差分廉价，参考绝对可信，还能画 2D 能量场；
- **含 sqrt + (d−L) 链式** → `∂E/∂x = Σ k(1−L/d)(x−ax)`，那个 `1−L/d` 里的 `1/d`
  正是 AI 手推最容易漏的项，也是靠近锚点时的奇异源；
- **有重复子表达式**（每个锚点的 d_i）→ 第 6 周的 `cse()` 有活可干，能量可复用。

`energy.h` 把锚点/L/k 写成 `constexpr` 常量，这样生成代码的签名就是干净的
`(const double* p, double* out)`，不用把一堆参数塞进函数入参。

## 二、两个有限差分工具：金标准的两种精度

```cpp
fd_gradient(p, g, h=1e-6);                     // 中心差分能量 → 参考梯度
fd_hessian_from_grad(grad, p, H, h=1e-5);      // 中心差分"解析梯度" → 参考 Hessian
```

关键设计：**Hessian 的参考不是二阶差分能量，而是一阶差分"已验证的解析梯度"**。
二阶差分能量 `(E(p+h)−2E(p)+E(p−h))/h²` 的舍入误差 ~ eps/h²，h=1e-5 时高达 1e-6，
根本没法卡 Hessian。而对解析梯度做一次中心差分，误差只有 ~eps/h≈1e-11——稳两个量级。
所以 `test_hessian` 里先把 `compute_gradient` 当"给定基础设施"（周一三已单独验过它），
再 FD 它。**验证是有依赖顺序的：先钉死梯度，再用梯度钉 Hessian。**

## 三、test_gradient：三条测试各抓一类错

```text
Test 1  与中心差分双算，max 相对误差 < 1e-6   → 抓任何符号/系数/链式错（正确 ~1e-10）
Test 2  沿 −grad 走能量必须下降              → 抓"梯度整体反号"（Test 1 也会爆，双保险）
Test 3  几乎压到锚点上，打印 grad            → 观察 1/d 奇异，AI 声明了吗
```

Test 1 的相对误差判据用 `d/max(1,|a|,|b|)`——分母兜底 1.0，避免梯度分量接近 0 时
相对误差假性放大。这是数值测试的常见小陷阱：纯相对误差在零点附近不可靠。

Test 3 不判分是故意的：`1/d` 在 d→0 时发散是**能量本身的性质**，不是 bug。
harness 的职责是把这个奇异摆到你面前，逼你在 prompt 里要求 AI "先声明奇异清单"
（第 2 周的技能在这里复用）。

## 四、test_hessian：对称性是"免费的强约束"

解析 Hessian 必然 `Hxy==Hyx`。这条对称性**任何正确实现都满足、任何漏项/错项都极易破坏**，
且判据可以卡到 1e-12（不是数值近似，是代数恒等）。Test 2 就用它当第二道闸：
Test 1（对 FD）抓量级错，Test 2（对称）抓"上下三角填得不一致"。

Test 3 打印每个点的 det/trace 判局部凸性。有意思的是采样点里
`p=(3.5,0.5)` 会显示**非正定**（det<0）——它离能量极小远、落在了鞍状区。
这不是错误，而是给你一个真实提醒：**牛顿法直接用这个 H 求解会翻车**，
非正定处要加阻尼或特征值截断。SymPy 能给你 H，但"H 不正定时怎么办"是你的活。

## 五、能量场 SVG：数值之外的第二只眼

`EnergySVG` 把能量画成热力图、把 **−梯度画成黑箭头**（下坡方向），锚点用红圈标出。
它接收的梯度回调就是你 PASTE 的 `compute_gradient`：

- 占位桩（返回 0）→ 箭头全部消失，一眼看出"还没接上"；
- 正确实现 → 箭头齐刷刷汇向暖色能量盆地；
- 符号写反 → 箭头**背离**盆地向外发散——比读数字更快发现方向错了。

这延续了整套课程的信条：**数值给你 PASS/FAIL，图形给你直觉**，两者互为冗余。

## 六、本周的元规律

第 5 周的口号是"永远让 AI 写脚本，永远跑一遍验证"。落到代码上，"跑一遍验证"= 有限差分对撞。
AI 把推导从半小时压到 30 秒，但**它不是最终裁判，中心差分才是**。
你要建立的不是"相信 AI 的推导"，而是"有一套自动裁判，让信不信 AI 都无所谓"。
